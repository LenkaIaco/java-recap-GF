equals() by default checks object equality(same reference + same type(instanceOf)+ same content), however in String, it is overwritten to check the content equality.
Equals() does NOT check for hashcode anywhere. It is only a contract that is recommended to be enforced while overwriting equals method to also overwrite hashCode(): 
 o1.equals(o2) -> o1.hashCode()==o2.hashCode()  
(while vice-versa works differently: hashCode== does not guarantee equals()=true)
HOWEVER, even if not checked by java's equals() functions, hashcode-based collections(HashMap, LinkedHashMap(extends HM) , HashSet, LinkedHashSet(extends HS), ConcurrentHashMap) assume that contract is kept and might behave incorrectly if the object used does not have the proper equals() method overwrite followed by hashCode()
