----------ConcurrentModificationException----------------------
While iterating an enhanced for loop(based on Iterator), or an Iterator, you CANNOT STRUCTURALLY modify(change SIZE) the underlying iterated Collection.
	- With for-each loops, this means you cannot use them for adding or removing!
	- While Iterator class can be used with remove() function, because java's Iterator is designed to be used to modify the properties of the underlying collection while removing(At the same time than removing elements, also modifying at the same time the 'expectedModCount' property, whose mismatch against 'modcount' is the cause of ConcurrentModificationException), so that the collection does not report a mismatch between the elements and not throw ConcurrentModificationException:

Iterator<Integer> itr = list.iterator();

while (itr.hasNext()){
itr.next();		//you can call next() multiple times/iteration, however without checking hasNext(), you risk hitting the NoSuchElementException when itr has run out of elements
itr.remove();		// you NEED to FIRST call next() and AFTER you can call remove() - otherwise: IllegalStateException !

}

- Before Java 14: must insert 'break' or 'return' for no fall-through:
	switch(String s){
	case "Saturday": System.out.println("Weekend starts");  | case "Saturday": return "Weekend starts";
	break; 
	default: System.out.println("Working week");
	}

	switch(number){
	case 1, 3, 5, 7, 9 -> System.out.println("odd number");
	case 2, 4, 6, 8, 10 -> System.out.println("even number");
	default -> System.out.println("bigger number");
	}
